## 如何使用 AI 帮你写代码（必读！）

本指南的核心使用方式：**你做指挥官，AI 做执行者**。

### 什么情况会用到 AI？

本文档前几步（第 0~3 步）会给你**完整的代码**，你直接复制粘贴到 VS Code 就行。

但从**第 4 步开始**，代码量急剧膨胀（游戏大厅、游戏引擎、WebSocket、AI 集成等），如果把完整代码全写出来，这个文档会变成几百页。所以从第 4 步起，我们采用新模式：

> **你告诉 AI 要做什么 → AI 写出代码 → 你把代码复制到文件里 → 验证**

本文档会给你**现成的 Prompt（提示词）**，你只是做"复制粘贴 Prompt 到 AI 对话窗口"这一步。

### 具体操作流程（3 步）

```
Step 1: 看本文档的"AI Prompt 框"
        ↓
Step 2: 复制框内的全部文字
        ↓
Step 3: 粘贴到 AI 对话窗口（Trae / Cursor / ChatGPT / Claude），按回车
        ↓
Step 4: AI 返回代码 → 复制代码 → 粘贴到 VS Code 对应的文件
```

### 什么样的 AI 工具可以用？

| 工具 | 推荐程度 | 说明 |
|------|---------|------|
| **Trae IDE**（你在用的） | ★★★★★ | 内置 AI，直接对话即可 |
| **Cursor** | ★★★★★ | 内置 AI，和 Trae 类似 |
| **ChatGPT / Claude** | ★★★★ | 需要手动复制粘贴代码 |
| **GitHub Copilot** | ★★★ | 自动补全为主，不支持长对话 |

> 如果你用的就是 **Trae IDE**，在这里按 `Ctrl+I`（或点击 AI 图标）打开对话窗口，把 Prompt 粘贴进去就行。

### Prompt 说明

每个 AI Prompt 框都包含：

- **你的项目背景**（告诉 AI 技术栈和目录结构）
- **你要做什么**（具体任务描述）
- **参考文档位置**（让 AI 自己去看架构文档里的详细设计）
- **输出要求**（告诉 AI 代码放哪个文件、用什么格式）

> ⚠️ **重要**：Prompt 里会引用架构文档的章节号（比如"参考 01-前端架构方案.md 5.3 节"）。AI 会自动去读那些文档，你不需要手动复制。

### 文档体系说明

本指南配合以下 4 份设计文档使用，每份负责不同层面：

| 文档 | 用途 | AI 什么时候需要读它 |
|------|------|-------------------|
| `01-前端架构方案.md` | 技术选型、路由、Store 设计、WebSocket 协议 | 前端所有步骤 |
| `02-后端架构方案.md` | DB Schema、状态机设计、Dify 集成策略、部署 | 后端所有步骤 |
| `03-API接口设计文档.md` | REST/WS 接口的请求/响应格式、错误码 | 前后端对接步骤 |
| `05-前端UI设计文档.md` | **每个页面的 ASCII 布局图 + px 尺寸 + 颜色 + 动画规格 + 交互流程** | 前端 UI 实现步骤（从第 3 步开始） |

> 简单说：架构文档定"做什么"，UI 设计文档定"长什么样"，API 文档定"怎么通信"。AI 三个都读了才能写出准确的代码。

interface StoredUser {

  nickname: string
  passwordHash: string
  level: number
  exp: number

  wins: number
  mvpCount: number
  createdAt: string

const users: Map<string, StoredUser> = new Map()

export const memoryDB = {

#### 1.5.2 修改 auth.service.ts

用 VS Code 打开 `E:\Code\UndercoverGame\server\src\modules\auth\auth.service.ts`，**把文件内容全部替换为以下代码**：

```typescript
import bcrypt from 'bcryptjs'
import jwt from 'jsonwebtoken'
import crypto from 'node:crypto'
import { AppError, ErrorCodes } from '../../shared/errors.js'
import { getEnv } from '../../config/env.js'
import { memoryDB } from '../../db/memory.js'
import type { RegisterInput, LoginInput } from './auth.schema.js'

const env = getEnv()

function signAccessToken(userId: string): string {
  return jwt.sign({ sub: userId }, env.JWT_SECRET, {
    expiresIn: env.ACCESS_TOKEN_EXPIRY,
  })
}

function signRefreshToken(userId: string): string {
  return jwt.sign({ sub: userId }, env.JWT_REFRESH_SECRET, {
    expiresIn: env.REFRESH_TOKEN_EXPIRY,
  })
}

function verifyToken(token: string, secret: string): { sub: string } {
  try {
    return jwt.verify(token, secret) as { sub: string }
  } catch {
    throw new AppError(ErrorCodes.TOKEN_EXPIRED, 'Token 无效或已过期', 401)
  }
}

class AuthService {
  async register(input: RegisterInput) {
    const { username, password, nickname } = input
    const existingUser = memoryDB.findByUsername(username)
    if (existingUser) {
      throw new AppError(ErrorCodes.USERNAME_TAKEN, '用户名已被注册', 409)
    }
    const passwordHash = await bcrypt.hash(password, 12)
    const userId = crypto.randomUUID()
    const user = memoryDB.createUser({
      id: userId,
      username,
      nickname: nickname || username,
      passwordHash,
      level: 1,
      exp: 0,
      totalGames: 0,
      wins: 0,
      mvpCount: 0,
      createdAt: new Date().toISOString(),
    })
    return {
      user: { id: userId, username, nickname: nickname || username, level: 1, exp: 0 },
      accessToken: signAccessToken(userId),
      refreshToken: signRefreshToken(userId),
    }
  }

  async login(input: LoginInput) {
    const { username, password } = input
    const user = memoryDB.findByUsername(username)
    if (!user) {
      throw new AppError(ErrorCodes.WRONG_CREDENTIALS, '用户名或密码错误', 401)
    }
    const valid = await bcrypt.compare(password, user.passwordHash)
    if (!valid) {
      throw new AppError(ErrorCodes.WRONG_CREDENTIALS, '用户名或密码错误', 401)
    }
    return {
      user: { id: user.id, username: user.username, nickname: user.nickname, level: user.level, exp: user.exp },
      accessToken: signAccessToken(user.id),
      refreshToken: signRefreshToken(user.id),
    }
  }

  async refresh(refreshToken: string) {
    const payload = verifyToken(refreshToken, env.JWT_REFRESH_SECRET)
    return {
      accessToken: signAccessToken(payload.sub),
      refreshToken: signRefreshToken(payload.sub),
    }
  }

  verifyAccessToken(token: string): { sub: string } {
    return verifyToken(token, env.JWT_SECRET)
  }
}

export const authService = new AuthService()
```

#### 1.5.3 修改 auth.guard.ts

用 VS Code 打开 `E:\Code\UndercoverGame\server\src\modules\auth\auth.guard.ts`，**把文件内容全部替换为以下代码**：

import type { FastifyRequest, FastifyReply } from 'fastify'
import { ErrorCodes } from '../../shared/errors.js'
import { authService } from './auth.service.js'

export async function authGuard(
  request: FastifyRequest,
  reply: FastifyReply,
): Promise<void> {
  const authHeader = request.headers.authorization
  if (!authHeader?.startsWith('Bearer ')) {
    reply.status(401).send({
      code: ErrorCodes.UNAUTHORIZED,
      message: '未提供认证令牌',
      data: null,
    })
    return
  }

  const token = authHeader.slice(7)
  try {
    const payload = authService.verifyAccessToken(token)
    ;(request as any).userId = payload.sub
  } catch {
    reply.status(401).send({
      code: ErrorCodes.TOKEN_EXPIRED,
      message: 'Token 无效或已过期',
      data: null,
    })
  }
}
```

import { ofetch } from 'ofetch'

const API_BASE = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000/api'

export const api = ofetch.create({
  baseURL: API_BASE,
  headers: { 'Content-Type': 'application/json' },
  onResponseError({ response }) {
    // 统一错误处理
    const body = response._data
    if (body?.message) {
      console.error('[API Error]', body.message)
    }
  },
})

// 请求拦截器：自动注入 Token
api.interceptors.request.use((options) => {
  const token = localStorage.getItem('accessToken')
  if (token) {
    options.headers = {
      ...options.headers,
      Authorization: `Bearer ${token}`,
    }
  }
  return options
})

### 3.3 实现认证 Store

打开 `src/stores/auth.ts`，写入完整的状态管理逻辑：

```typescript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import * as authService from '@/services/authService'

export interface User {
  id: string
  username: string
  nickname: string
  level: number
  exp: number
  avatarUrl?: string
}

export const useAuthStore = defineStore('auth', () => {
  const user = ref<User | null>(null)
  const token = ref<string | null>(localStorage.getItem('accessToken'))
  const refreshToken = ref<string | null>(localStorage.getItem('refreshToken'))

  const isAuthenticated = computed(() => !!token.value)

  async function login(username: string, password: string) {
    const res: any = await authService.login({ username, password })
    if (res.code === 0) {
      user.value = res.data.user
      token.value = res.data.accessToken
      refreshToken.value = res.data.refreshToken
      localStorage.setItem('accessToken', res.data.accessToken)
      localStorage.setItem('refreshToken', res.data.refreshToken)
    }
    return res
  }

  async function register(username: string, password: string, nickname?: string) {
    const res: any = await authService.register({ username, password, nickname })
    if (res.code === 0) {
      user.value = res.data.user
      token.value = res.data.accessToken
      refreshToken.value = res.data.refreshToken
      localStorage.setItem('accessToken', res.data.accessToken)
      localStorage.setItem('refreshToken', res.data.refreshToken)
    }
    return res
  }

  function logout() {
    user.value = null
    token.value = null
    refreshToken.value = null
    localStorage.removeItem('accessToken')
    localStorage.removeItem('refreshToken')
  }

  async function fetchProfile() {
    const res: any = await authService.getMe()
    if (res.code === 0) {
      user.value = res.data.user
    }
  }

  return { user, token, refreshToken, isAuthenticated, login, register, logout, fetchProfile }
})

### 5.1 实现游戏状态机（完整代码框架）

打开 `E:\Code\UndercoverGame\server\src\modules\game\game.engine.ts`，**把文件内容全部替换为以下代码**：

```typescript
/**
 * game.engine.ts - 游戏状态机（基于 xstate v5）
 *
 * 状态流转：
 * WAITING → ROLE_ASSIGN → SPEAKING ⇄ VOTING → RESULT → FINISHED
 */
import { createMachine, assign } from 'xstate'
import type { Player, Speech, VoteRecord } from '../../types/game.js'
import { GAME_SLOTS, MAX_ROUNDS } from '../../config/constants.js'
import crypto from 'node:crypto'

// ─── 上下文类型 ───
interface GameContext {
  gameId: string
  civilianWord: string
  undercoverWord: string
  players: Player[]
  currentRound: number
  currentSpeakerIndex: number
  undercoverSlotIndex: number
  speeches: Speech[]
  votes: VoteRecord[]
  eliminatedPlayers: number[]
  maxRounds: number
}

// ─── 事件类型 ───
type GameEvent =
  | { type: 'START_GAME'; civilianWord: string; undercoverWord: string; players: Player[] }
  | { type: 'ROLES_ASSIGNED' }
  | { type: 'SPEECH_SUBMIT'; slotIndex: number; content: string }
  | { type: 'ALL_SPOKEN' }
  | { type: 'VOTE_CAST'; voterSlot: number; targetSlot: number }
  | { type: 'ALL_VOTED' }
  | { type: 'ELIMINATE' }
  | { type: 'CHECK_WINNER' }
  | { type: 'NEXT_ROUND' }
  | { type: 'GAME_OVER' }

// ─── 状态机定义 ───
export const gameMachine = createMachine({
  id: 'undercoverGame',
  initial: 'waiting',
  context: {} as GameContext,
  types: {} as {
    context: GameContext
    events: GameEvent
  },
  states: {
    waiting: {
      on: {
        START_GAME: {
          target: 'role_assign',
          actions: assign({
            gameId: () => crypto.randomUUID(),
            civilianWord: ({ event }) => event.civilianWord,
            undercoverWord: ({ event }) => event.undercoverWord,
            players: ({ event }) => event.players,
            currentRound: 0,
            speeches: () => [],
            votes: () => [],
            eliminatedPlayers: () => [],
          }),
        },
      },
    },

    role_assign: {
      entry: assign({
        // 随机选择 1 人为卧底
        undercoverSlotIndex: () => Math.floor(Math.random() * GAME_SLOTS),
        currentRound: ({ context }) => context.currentRound + 1,
      }),
      // 分配完成，自动进入发言阶段
      always: {
        target: 'speaking',
        actions: assign({
          players: ({ context }) =>
            context.players.map((p, i) => ({
              ...p,
              role: i === context.undercoverSlotIndex
                ? ('undercover' as const)
                : ('civilian' as const),
              word:
                i === context.undercoverSlotIndex
                  ? context.undercoverWord
                  : context.civilianWord,
            })),
          currentSpeakerIndex: 0,
        }),
      },
    },

    speaking: {
      // 收到发言 → 记录发言 → 轮到下一个人
      on: {
        SPEECH_SUBMIT: {
          actions: assign({
            speeches: ({ context, event }) => [
              ...context.speeches,
              {
                slotIndex: event.slotIndex,
                content: event.content,
                round: context.currentRound,
                isAI: false,
                timestamp: Date.now(),
              },
            ],
            currentSpeakerIndex: ({ context }) => {
              // 找下一个存活的发言人
              let next = context.currentSpeakerIndex + 1
              while (next < GAME_SLOTS) {
                if (!context.eliminatedPlayers.includes(next)) return next
                next++
              }
              return -1 // 所有人发言完毕
            },
          }),
        },
        ALL_SPOKEN: 'voting',
      },
    },

    voting: {
      on: {
        VOTE_CAST: {
          actions: assign({
            votes: ({ context, event }) => [
              ...context.votes,
              {
                voterSlot: event.voterSlot,
                targetSlot: event.targetSlot,
                timestamp: Date.now(),
              },
            ],
          }),
        },
        ALL_VOTED: 'eliminate',
      },
    },

    eliminate: {
      entry: assign({
        // 计算票数，淘汰最高票者；平票则无人淘汰
        eliminatedPlayers: ({ context }) => {
          const votesByTarget: Record<number, number> = {}
          for (const v of context.votes) {
            votesByTarget[v.targetSlot] = (votesByTarget[v.targetSlot] || 0) + 1
          }
          let maxVotes = 0
          let maxSlot = -1
          let isTie = false
          for (const [slot, count] of Object.entries(votesByTarget)) {
            const s = Number(slot)
            if (count > maxVotes) {
              maxVotes = count
              maxSlot = s
              isTie = false
            } else if (count === maxVotes) {
              isTie = true
            }
          }
          if (isTie || maxSlot === -1) return [...context.eliminatedPlayers]
          return [...context.eliminatedPlayers, maxSlot]
        },
      }),
      always: 'check_winner',
    },

    check_winner: {
      always: [
        // 卧底被淘汰 → 平民胜利
        {
          target: 'result',
          guard: ({ context }) =>
            context.eliminatedPlayers.includes(context.undercoverSlotIndex),
        },
        // 只剩 2 人且卧底存活 → 卧底胜利
        {
          target: 'result',
          guard: ({ context }) => {
            const alive = context.players.filter(
              (_, i) => !context.eliminatedPlayers.includes(i),
            )
            return (
              alive.length <= 2 &&
              !context.eliminatedPlayers.includes(context.undercoverSlotIndex)
            )
          },
        },
        // 达到最大轮数 → 卧底胜利
        {
          target: 'result',
          guard: ({ context }) => context.currentRound >= (context.maxRounds || MAX_ROUNDS),
        },
        // 还没结束 → 下一轮
        { target: 'next_round' },
      ],
    },

    next_round: {
      entry: assign({
        currentRound: ({ context }) => context.currentRound + 1,
        speeches: () => [],
        votes: () => [],
        currentSpeakerIndex: 0,
      }),
      always: 'speaking',
    },

    result: {
      always: 'finished',
    },

    finished: {
      type: 'final',
    },
  },
})
```

> **这段代码是游戏的核心逻辑**。它定义了一个状态机，游戏在不同状态间流转：
> - `waiting` 等待玩家开始 → `role_assign` 分配卧底身份 → `speaking` 轮流发言 → `voting` 投票 → `eliminate` 淘汰 → `check_winner` 判断胜负 → 要么下一轮要么结束。
> - 你可以先复制粘贴，不用完全理解。后续对接 WebSocket 时会逐步跑通。

  eliminatedPlayers: number[]
  undercoverSlotIndex: number

const rooms = new Map<string, GameRoom>()

export const roomManager = {
  createRoom(gameId: string, data: Partial<GameRoom>): GameRoom {
    const room: GameRoom = {
      gameId,
      status: 'waiting',
      civilianWord: data.civilianWord || '',
      undercoverWord: data.undercoverWord || '',
      players: data.players || [],
      currentRound: 0,
      eliminatedPlayers: [],
      undercoverSlotIndex: -1,
    }
    rooms.set(gameId, room)
    return room
  },

  getRoom(gameId: string): GameRoom | undefined {
    return rooms.get(gameId)
  },

  updateRoom(gameId: string, updates: Partial<GameRoom>): void {
    const room = rooms.get(gameId)
    if (room) Object.assign(room, updates)
  },

  removeRoom(gameId: string): void {
    rooms.delete(gameId)
  },
}

### 6.0 前置知识：RAG 与词库

> **为什么不用纯 LLM 生成词语？**
> 
> 纯 LLM 生成词语有三个致命问题：
> 1. **不可靠**：可能编造不存在或不合规的词对（如"电脑 vs 香蕉"毫无关联）
> 2. **难控制**：无法按难度筛选（简单/适中/困难），无法按主题分类
> 3. **成本高**：每次生成都消耗 Token，200+ 对词全部靠 LLM 生成成本大
> 
> **RAG（检索增强生成）方案**：
> 1. 提前将 200+ 对标注好难度和主题的真实词对上传到 Dify **知识库**
> 2. 每次需要词语时，知识库先按条件检索（如"难度：适中"）召回 10 对候选
> 3. LLM 只负责从 10 对候选中**随机选择 1 对**，不编造新词
> 4. 词库可持续扩展，词语来源可追溯

- 所有逻辑在一个应用内，方便调试

**缺点：**
- 工作流复杂度更高（分支多）
- 如果一处出问题，整个应用受影响
- 不如 3 个独立应用灵活

**分支逻辑：**
```
用户输入 type 字段
  ├── type = "词语生成"  → IF 分支：走词语生成工作流（知识检索 + LLM）
  ├── type = "语义描述"  → ELIF 分支：走 AI 发言逻辑
  └── type = "投票"      → ELSE 分支：走投票逻辑
```

**后端调用方式：**
只需维护 1 个 API Key，通过 `inputs.type` 字段区分功能：

```typescript
// 词语生成
POST /v1/chat-messages
{ inputs: { type: "词语生成" }, query: "难度：适中", ... }

// AI 发言
POST /v1/chat-messages
{ inputs: { type: "语义描述", currentWord: "苹果", personality: "logic" }, query: "其他玩家发言...", ... }

// AI 投票
POST /v1/chat-messages
{ inputs: { type: "投票", currentWord: "苹果", friends: "...", personality_: "logic" }, query: "所有发言...", ... }
```

> 建议：新手先用单一智能体方案快速跑通流程，后续再拆分为 3 个独立应用以获得更好的维护性。

## 附录A：Dify 调试常见问题

> 📖 问题来源和详细代码参见 `.trae/docs/guide/AI词语查询模块.md` 和 `.trae/docs/guide/RestTemplate发起客户端请求流程.md`

### A.1 401 Unauthorized

**原因**：API Key 格式错误。

**检查清单**：
1. API Key 是否以 `app-` 开头
2. Authorization header 格式是否为 `Bearer app-xxx`（**Bearer 后有一个空格**）
3. API Key 是否属于当前应用（每个 Dify 应用有独立的 Key）

### A.2 JSON 解析失败

**原因**：Dify 返回的 `answer` 字段不是纯净 JSON，可能包含：
- AI 的思考文字（如 "好的，我来生成... { ... }"）
- Markdown 代码块包裹（` ```json { ... } ``` `）
- 多余的换行和空格

**解决方案**：必须实现 `extractJsonFromAnswer()` 工具函数，兼容三种格式：
1. 检测 ` ``` ` 标记 → 剔除代码块包裹
2. 找到第一个 `{` 和最后一个 `}` → 截取中间内容
3. 纯净 JSON → 直接解析

> TypeScript 实现参考：将 `.trae/docs/guide/AI词语查询模块.md` 中的 Java 代码翻译为 TypeScript 即可。

### A.3 Content-Type 未设置导致解析失败

**原因**：请求头缺少 `Content-Type: application/json`，Dify 无法解析请求体。

**解决**：确保 ofetch 请求中包含 `headers: { 'Content-Type': 'application/json' }`。

### A.4 Dify 5 个固定入参

调用 Dify `/v1/chat-messages` 接口时，请求体必须包含 5 个字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `inputs` | 自定义业务参数 | `{ type: "词语生成" }` |
| `query` | 发给大模型的提示词 | `"难度：适中"` |
| `response_mode` | `blocking`（一次性返回）或 `streaming`（流式） | `"blocking"` |
| `conversation_id` | 会话 ID，空字符串 = 新建对话 | `""` |
| `user` | 用户标识，用于 Dify 统计 | `"abc-123"` |

### A.5 词语重复/不随机

**原因**：LLM 倾向选择列表第一项，导致总是返回同一对词。

**解决**：System Prompt 中明确要求 "**随机选择1对**，不要总是选固定位置的"，并在 JSON Schema 中启用强制输出格式约束。

### 13.0 安装数据库（部署前必须装）

之前 12 个步骤的开发全部用内存存储。现在要部署到公网了，需要装真正的数据库来持久化数据。

#### 安装 PostgreSQL 16

PostgreSQL 是用来存用户、游戏记录、排行榜等数据的数据库软件。

1. 打开浏览器，访问：**https://www.postgresql.org/download/windows/**
2. 点击 **"Download the installer"** 按钮
3. 在版本列表中选择 **PostgreSQL 16.x** 版本
4. 双击下载的安装包（`.exe` 文件）
5. 安装过程中会让你设置一个密码 — **记下来**（建议用简单的 `postgres`，不容易忘）
6. 端口那一页保持默认的 `5432`，不用改
7. 其他页面全部点 Next，直到安装完成
8. 安装完成后在开始菜单找到 **pgAdmin 4**，打开确认能连上

> 验证方法：打开 pgAdmin 4 → 输入你设置的密码 → 能看到数据库列表，就说明装好了。

#### 安装 Redis

Redis 是内存缓存数据库，用来加速排行榜查询和存储房间临时状态。

> Redis 官方没有 Windows 版本。用社区维护的版本就行。

1. 打开浏览器，访问：**https://github.com/tporadowski/redis/releases**
2. 找到最新版本，下载 **Redis-x64-xxx.msi** 文件
3. 双击 `.msi` 文件安装，一路点 Next
4. 安装完成后 Redis 会自动在后台启动（你看不到窗口，这是正常的）

> 验证方法：打开 PowerShell，输入 `redis-cli ping`，如果返回 `PONG` 就说明 Redis 在运行。

#### 修改 .env 配置数据库连接

数据库装完后，还要让后端知道怎么连。用 VS Code 打开 `server/.env`：

```
DATABASE_URL=postgresql://postgres:你的密码@localhost:5432/undercover_game
REDIS_URL=redis://localhost:6379
```

把"你的密码"替换成安装 PostgreSQL 时设置的那个密码。比如密码是 `postgres`：

```
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/undercover_game
```

> 我运行 npm run dev 后出现了以下错误，请帮我修复：
> [粘贴红色报错信息]
> ```

大多数情况下 AI 能自己修复。

### 常见问题速查

| 报错信息 | 原因 | 修复方法 |
|----------|------|---------|
| `Cannot find module '@/...'` | 路径别名未配置 | 检查 0.9 节，确认 vite.config.ts 中有 @ 别名 |
| `Cannot find module '../../db/memory.js'` | 后缀名错误 | 导入路径改成 `.js` 后缀（TypeScript ESM 要求） |
| `Type 'xxx' is not assignable to type 'yyy'` | 类型不匹配 | 把报错粘贴给 AI："请修复类型错误" |
| `xstate.createMachine is not a function` | 导入方式错误 | 检查是否 `import { createMachine } from 'xstate'` |
| `WebSocket connection to 'ws://...' failed` | 后端未启动或端口错误 | 确认 npm run dev 在后端目录运行 |
| `401 Unauthorized` | Token 过期或未传 | 重新登录，确认 localStorage 中有 accessToken |
| `ref(...) is not defined` | 忘记 `import { ref } from 'vue'` | 把报错粘贴给 AI |
| 组件显示空白 | 组件名大小写不对 | 检查 import 的组件名和 template 中使用的标签名一致 |

### 系统性排查流程

当你不知道哪里错了，按这个顺序排查：

```
1. 切到 VS Code 的"终端"面板，看有没有红色报错
   - 有报错 → 复制粘贴给 AI
   - 没报错 → 下一步

2. 打开浏览器 http://localhost:5173，按 F12 打开开发者工具
   - Console 标签：看有没有红色报错 → 有就粘贴给 AI
   - Network 标签：看 API 请求状态
     - 红色(4xx/5xx) → 检查请求 URL 和参数
     - pending(一直转) → 后端没返回，检查后端日志

3. 页面能打开但布局不对
   - 按 F12 → Elements 标签 → 检查 HTML 结构
   - 检查 CSS class 是否正确应用
   - 把"期望效果描述 + 当前截图描述"粘贴给 AI
```

### 重点：游戏页面（第 7 步）常见问题

第 7 步是最容易出问题的一步，额外提供几个针对性修复 Prompt：

> 🤖 **WebSocket 连不上：**
> ```
> 我的 WebSocket 连接 ws://localhost:3000/ws 一直 connecting 状态。
> 后端 npm run dev 在运行，/api/health 能正常访问。
> 请帮我检查 useWebSocket.ts 的连接逻辑。
> ```

> 🤖 **AI 发言打字机效果不显示：**
> ```
> AI 发言的 ai_speech_chunk 消息收到了（console 能看到），但页面上没有逐字显示动画。
> 请检查 SpeechArea.vue 中流式文本的渲染逻辑。
> ```

> 🤖 **投票后页面没反应：**
> ```
> 我点击投票后发送了 cast_vote 消息，但页面没有更新票型。
> 请检查 useWebSocket.ts 中对 vote_update 消息的处理逻辑。
> ```
