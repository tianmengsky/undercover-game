# AI谁是卧底大乱斗 — 前端架构方案

---

## 一、项目概述

### 1.1 项目定位
基于 Vue 3 的「谁是卧底」社交推理游戏 SPA，玩家与多个 AI Agent（通过 Dify 大模型驱动）同场竞技。核心体验：实时发言、投票推理、AI 人设扮演。

### 1.2 技术选型

| 类别 | 技术 | 说明 |
|------|------|------|
| 框架 | Vue 3.4+ (Composition API) | 主力框架 |
| 语言 | TypeScript 5.x | 类型安全 |
| 构建 | Vite 6.x | 极速 HMR |
| 路由 | Vue Router 4.x | SPA 路由 |
| 状态管理 | Pinia 2.x | 轻量响应式状态 |
| 实时通信 | socket.io | 游戏信令，自动重连 + 房间管理 |
| HTTP 客户端 | ofetch | 轻量、原生 TS 类型推断、Vite 生态原生 |
| UI 组件库 | 自定义组件 + CSS Variables | 清爽主题，灵活可控 |
| 动画 | @vueuse/motion + CSS View Transitions | 淘汰、翻牌、过渡动画 |
| TTS | Web Speech API / 后端 TTS 服务 | 语音朗读 |
| 表单校验 | 自定义 composable | 轻量 |
| 测试 | Vitest + Playwright | 单测 + E2E |
| PWA | vite-plugin-pwa | 离线缓存 + 可安装 |

### 1.3 主题色方案


## 三、路由设计

## 四、核心状态设计（Pinia Store）

> **职责划分原则**：`stores/` 只放响应式数据容器（状态+getter），`composables/` 负责业务流程编排（调用 store action + service API + WebSocket）。

### 4.1 authStore

```typescript
interface AuthState {
  user: User | null
  token: string | null
  isAuthenticated: boolean
}
// actions: login(), register(), logout(), fetchProfile()
```

### 4.2 gameStore（核心）

```typescript
interface GameState {
  gameId: string
  status: GameStatus          // 'waiting' | 'role_assign' | 'speaking' | 'voting' | 'result' | 'finished'
  currentRound: number
  maxRounds: number
  players: Player[]           // 6人
  currentSpeakerIndex: number
  civilianWord: string
  undercoverWord: string
  isUndercoverEliminated: boolean
  voteResults: VoteRecord[]
  remainingTime: number       // 倒计时秒数
  settings: GameSettings
}

interface Player {
  id: string
  name: string
  avatar: string
  isAI: boolean
  aiPersona?: AIPersona       // AI 人设
  isAlive: boolean
  role?: 'civilian' | 'undercover'
  word?: string
  isCurrentSpeaker: boolean
  hasVoted: boolean
}
```

### 4.3 chatStore

```typescript
interface ChatState {
  messages: ChatMessage[]
  currentRoundMessages: ChatMessage[]
}
// ChatMessage: { id, playerId, playerName, content, round, timestamp, type }
```

### 4.4 achievementsStore

```typescript
interface AchievementsState {
  allAchievements: AchievementDef[]
  unlockedIds: Set<string>
  newlyUnlocked: AchievementDef[]  // 本局新解锁（触发弹窗）
}
```

## 五、页面设计详述

### 5.1 首页 (HomeView)
- 展示游戏标题 Logo、Slogan
- 规则简介（卡片式3步说明）
- 「开始游戏」和「排行榜」按钮
- 已登录显示用户名 + 等级徽章

### 5.2 登录/注册页 (LoginView / RegisterView)
- 居中卡片布局
- 用户名 + 密码（注册加确认密码 + 昵称）
- 表单校验 + 错误提示
- 切换登录/注册的链接

### 5.3 游戏大厅 (GameLobbyView)
- **角色配置区域**：6个角色槽位
  - 1个玩家槽位（固定为当前用户，可自定义昵称）
  - 5个 AI Agent 槽位，每个可为 AI 选择「人设」
  - AI 人设选项：`默认`、`幽默达人`、`逻辑大师`、`萌新小白`、`文艺青年`、`暴躁老哥`
- **词语生成区域**：
  - 手动输入：平民词 + 卧底词
  - **AI 生成（推荐）**：调用 Dify RAG 知识库工作流，从 200+ 对真实标注词库中按难度筛选，由 LLM 随机选择一对。不是纯 LLM 凭空生成，词语来源可追溯、质量可控
  - 难度选择器：`简单` / `适中` / `困难`（控制 RAG 知识库检索的难度标签过滤）
  - 「随机换一组」按钮
- **游戏规则摘要**：侧边栏或折叠面板
- 「开始游戏」按钮（带加载状态）

### 5.4 游戏主界面 (GameView)
布局设计：**三栏 + 顶部信息栏**

```
┌──────────────────────────────────────────────┐
│  🔵 第 3/5 轮  │  存活 5/6 人  │  🔊语音  │  ⚙设置 │
├──────────┬───────────────────────┬───────────┤
│          │                      │           │
│  玩家    │     💬 发言区域       │  玩家     │
│  列表    │                      │  列表     │
│  (左侧)  │   ┌──────────────┐   │  (右侧)   │
│          │   │  发言气泡流    │   │           │
│  👤 你   │   │  (滚动)      │   │  👤 AI_1 │
│  👤 AI_1 │   └──────────────┘   │  👤 AI_2 │
│  👤 AI_2 │                      │  👤 AI_3 │
│          │  ┌───发言输入框───┐   │           │
│          │  │ 输入你的描述.. │   │           │
│          │  │     [发送]     │   │           │
│          │  └───────────────┘   │           │
│          │                      │           │
└──────────┴───────────────────────┴───────────┘
```

功能区说明：
- **顶部信息栏**：轮次进度条、存活计数、TTS开关、设置按钮
- **左侧玩家列表**：圆角头像 + 状态标签（发言中/已发言/已淘汰），轮到发言时高亮
- **中央发言区**：
  - 可滚动发言气泡流（最新消息自动滚到底部）
  - AI 发言带人设标签 + 打字动画
  - 玩家发言输入框 + 发送按钮
- **右侧玩家列表**：6人时左右各3人，布局对称
- **投票阶段**：中央区域切换为投票面板，展示存活玩家头像，点击投票
- **淘汰动画**：玩家被投票淘汰时播放
- **词语揭示**：卧底被淘汰或存活到最后时展示双方词语

### 5.5 结算页 (ResultView)
- 胜负展示动画（平民胜利 / 卧底胜利）
- 双方词语揭示
- 各玩家身份揭露
- 本局 MVP（投票最准 / 隐藏最深）
- 经验值获取 + 等级变化动画
- 「再来一局」「返回大厅」按钮

### 5.6 个人中心 (ProfileView)
- 头像 + 昵称 + 等级 + 经验条
- 战绩统计：总局数、胜率、MVP 次数
- 历史对局列表（分页）
- 各人设使用频率雷达图（可选加分项）

### 5.7 排行榜 (LeaderboardView)
- 全服排行：按积分/胜率排序
- 筛选：总榜 / 周榜 / 月榜
- 列表项：排名、头像、昵称、等级、胜场、积分
- 当前用户排名高亮

### 5.8 游戏回放页 (ReplayView)（创新加分项）
- 时间轴播放器：横向进度条 + 播放/暂停/倍速
- 发言事件按时间线排列，点击跳转到对应时刻
- 每条发言气泡显示头像、人设标签、时间戳
- 投票阶段展示票型分布动画
- 淘汰节点播放角色揭露特效
- 关键节点插入 AI 心理旁白："卧底此时心里在想..."
- 底部「分享回放」按钮：生成分享链接

### 5.9 结算页增强 — 词语关键词云
- 使用 `vue-wordcloud` 或 Canvas 绘制
- 平民发言词用蓝色，卧底发言词用橙色
- 卧底的"危险词"（疑似暴露身份的用词）高亮红色
- 玩家可直观看到 AI 的推理路径和卧底的破绽

### 5.10 成就展示页 (AchievementView)
- 网格布局展示所有成就（灰色=未解锁，彩色=已解锁）
- 已解锁成就显示解锁日期和对局信息
- 进度条类成就（如"累计 50 局"）显示当前进度
- 新解锁成就全局弹窗动画（结算后、大厅进入时）

### 5.11 AI 人设工坊 (PersonaWorkshopView)（创新加分项）
- 输入框：用自然语言描述一个 AI 性格
- 示例提示："一个总爱引用古诗词的老教授" / "一个说话阴阳怪气的侦探"
- 调用 Dify `persona-creator` 生成 System Prompt 预览
- 确认后保存，纳入可选人设列表
- 公共工坊：浏览其他玩家创建的人设，点赞/选用
- 限制：每人每月最多创建 3 个

## 六、socket.io 通信设计

### 6.1 连接管理 (`useWebSocket`)
- 基于 socket.io-client，自动连接 / 断线重连（指数退避：1s, 2s, 4s, 8s, 16s，最多 5 次，socket.io 内置）
- 心跳保活（socket.io 内置 ping/pong）
- 游戏 ID 绑定（连接握手 `auth` 参数传递 gameId）
- **重连状态恢复**：重连成功后服务端自动发送完整 `game_state` 快照 → `gameStore` 覆盖刷新 → UI 无缝恢复

### 6.2 消息类型

| 客户端 → 服务端 | 说明 |
|----------------|------|
| `player_speech` | 玩家发言（交给 orchestrator resolver） |
| `cast_vote` | 投票（交给 orchestrator resolver） |
| `toggle_tts` | 切换语音朗读 |

| 服务端 → 客户端 | 说明 |
|----------------|------|
| `game_state` | 完整游戏状态快照（连接/重连时） |
| `game_started` | 游戏正式开始 |
| `phase_change` | 阶段切换（speaking / voting），含 `phase` + `round` |
| `current_speaker` | 当前发言人（AI 或人类） |
| `request_speech` | 轮到人类玩家发言 |
| `ai_speech_start` | AI 开始发言 |
| `ai_speech_chunk` | AI 发言流式内容块（打字机效果） |
| `ai_speech_end` | AI 发言结束（含完整内容） |
| `player_speech_broadcast` | 人类玩家发言广播 |
| `speech_timeout` | 发言超时跳过 |
| `current_voter` | 当前投票人 |
| `request_vote` | 轮到人类玩家投票（含候选人列表） |
| `vote_cast` | 投票结果（每票即时广播） |
| `vote_timeout` | 投票超时（自动随机投） |
| `player_eliminated` | 淘汰结果（含是否卧底） |
| `elimination_tie` | 平票无人淘汰 |
| `next_round` | 进入下一轮 |
| `game_over` | 游戏结束（含胜负、词语揭晓、玩家状态） |
| `game_error` | 游戏运行异常 |
| `error` | 通用错误信息 |
| `achievement_unlocked` | 成就解锁通知 |

## 八、性能优化

1. **代码分割**：路由级懒加载 `() => import('./views/XxxView.vue')`
2. **虚拟滚动**：排行榜/历史对局列表使用 `vue-virtual-scroller`（如数据量大）
3. **WebSocket 消息节流**：投票等高频操作前端防抖
4. **静态资源**：图片压缩、SVG 内联关键图标
5. **CSS**：使用 `scoped` + CSS Modules 避免样式泄漏，关键动画使用 `transform/opacity` 触发 GPU 加速

## 十、PWA 支持

使用 `vite-plugin-pwa` 实现：

- **离线缓存**：静态资源（JS/CSS/图片）+ 首页预缓存，游戏进行中不依赖网络（降级模式使用本地规则引擎替代 Dify）
- **可安装**：manifest.json 配置，支持添加到桌面
- **更新提示**：检测到新版本时弹窗提示刷新
- **缓存策略**：
  - API 请求：Network First（网络优先，失败回退缓存）
  - 静态资源：Cache First（缓存优先，后台更新）
  - 游戏状态：仅 WebSocket，不缓存

**离线降级模式**：当网络不可用时，提供纯本地规则引擎的"练习模式"——AI 发言从本地模板库生成，投票基于规则算法，让玩家可以随时练习。此模式不计入战绩统计。
