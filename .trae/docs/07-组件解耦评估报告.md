## 二、建议抽离实现（12 个）

### 2.1 通用组件层（6 个）

#### `components/common/AppButton.vue` ⭐ 高收益

| 当前 | `btn-primary`、`btn-big`、`btn-text-sm`、`btn-submit` CSS 分散 8+ 处 |
|------|------|
| Props | `variant: 'primary'\|'secondary'\|'danger'\|'ghost'`、`size: 'sm'\|'md'\|'lg'`、`disabled`、`loading` |
| 收益 | 消除 8 处重复 CSS + 统一 disable/loading 状态 |
| 影响 | 零 — 用 `<AppButton variant="primary">` 替换 `<button class="btn-primary">` |

```
抽取前（8 处各自定义）：
.btn-primary   { background: primary; }    // GameLobbyView
.btn-submit    { width: 100%; height: 44px; } // LoginView
.btn-big       { min-width: 260px; }        // ResultView
...

抽取后（1 处统一定义）：
<AppButton variant="primary" size="lg" :loading="starting">开始游戏</AppButton>
<AppButton variant="primary" size="md" :disabled="!canSubmit">登录</AppButton>
```

#### `components/common/AppInput.vue` ⭐ 高收益

| 当前 | `.form-group` 模式在 LoginView / RegisterView / PersonaWorkshopView 中重复 10+ 次 |
|------|------|
| Props | `label`、`modelValue`、`type`、`placeholder`、`error`、`required`、`maxlength` |
| 收益 | 消除 ~60 行重复 CSS + 统一 label/error/required 布局 |
| 影响 | 零 — 纯展示组件，无耦合 |

```
抽取前（每处 ~8 行）：
<div class="form-group">
  <label>用户名 <span class="required">*</span></label>
  <input v-model="username" placeholder="..." />
  <p v-if="usernameError" class="field-error">{{ usernameError }}</p>
</div>

抽取后（每处 ~2 行）：
<AppInput v-model="username" label="用户名" placeholder="..." :error="usernameError" required />
```

#### `components/common/AppModal.vue` ⭐ 高收益

| 当前 | GameView 的淘汰动画 + 阶段过渡倒计时都是内联 overlay+Transition |
|------|------|
| Props | `visible`、`title?`、`closable`、`contentClass?` |
| Slots | `default`（内容）、`footer?` |
| 收益 | 消除 2 处 ~50 行重复的 overlay/transition/close 逻辑 |
| 影响 | 零 — 纯 UI 组件 |

#### `components/common/AppToast.vue` + `composables/useToast.ts` ⭐ 高收益

| 当前 | 全局 `alert()` 5 处 + GameView 内联 toast 2 处 |
|------|------|
| 方案 | `useToast()` 提供 `toast.success(msg)` / `toast.error(msg)`，AppToast 渲染队列 |
| 收益 | 替代所有 `alert()`，统一视觉风格，支持自动消失+堆叠 |
| 影响 | 零 — 新增功能，不破坏现有逻辑 |

#### `components/common/AppLoading.vue`

| 当前 | `v-if="loading">加载中...</div>` 散落 6+ 处 |
|------|------|
| Props | `text?`、`variant: 'inline'\|'overlay'` |
| 收益 | 统一 Loading 视觉 + 支持全屏遮罩加载 |
| 影响 | 零 — `<AppLoading v-if="loading" />` 直替换 |

## 三、无需抽离（18 个）

这些文件功能已在 View 中内联实现。抽离会增加 props 传递、事件定义和样板代码，**负面影响 > 收益**。

| 文件 | 原因 |
|------|------|
| `components/auth/LoginForm.vue` | 4 个 input+button，抽离要传 6 个 props + 1 个 emit |

| `components/game/GameBoard.vue` | 三栏布局在 GameView 中和 WS 消息处理强耦合 |
| `components/game/GameLobby.vue` | 整个大厅和 createGame/startGame 强耦合 |
| `components/game/GameSetup.vue` | 词语设置面板和 lobby 的 wordMode 绑定在一起 |
| `components/game/GameResult.vue` | 结算页 600 行，但所有数据来自 gameStore，是一个视图而非组件 |
| `components/game/RoleAssign.vue` | 当前只是一个 WS 消息触发 toast |
| `components/game/PersonaSelector.vue` | 一个 `<select>` 标签不值得组件化 |
| `components/game/SpeechBubble.vue` | 已被 SpeechArea.vue 替代 |
| `components/persona/PersonaWorkshop.vue` | 整个工坊页逻辑集中，不应该拆 |
| `components/replay/ReplayViewer.vue` | 播放控制只有 3 个按钮 |
| `components/replay/ReplayTimeline.vue` | 事件列表的 5 种类型条件渲染 IS 这个组件的全部价值 |
| `components/replay/ReplaySpeechBubble.vue` | 和 currentIndex 强绑定 |
| `components/profile/PlayerProfile.vue` | 需要 ProfileView 先完成 |
| `components/profile/GameHistory.vue` | 同上 |
| `components/common/AppAvatar.vue` | 单字符 emoji 就是头像 |
| `components/common/AppBadge.vue` | `<span class="tag">` 1 行 |
| `components/common/AppCard.vue` | `<div class="card">` 语义化 div 最好 |

## 六、不做的事

1. **不抽 LoginForm/RegisterForm** — 表单输入绑定和校验变量多，抽离传递 6+ props/emits 不值得
2. **不抽 GameBoard/GameLobby** — 布局和游戏状态深度耦合
3. **不抽 ReplayViewer/ReplayTimeline** — 简单按钮/事件列表不值得独立文件
4. **不抽 AppAvatar/AppBadge/AppCard** — 每个仅 1-2 行，组件化增加复杂性
5. **不删任何预留文件** — 保留文件结构，内容留 TODO 无负面影响
